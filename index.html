Match Engine Technical Workflow

ResuMatch ATS operates as an interactive debugger, breaking both documents into vector chunks and comparing alignments locally before running AI evaluations.

1

Text Extraction

The file uploader extracts plain text client-side using PDF.js and Mammoth.js. Text formatting, spacing, and page elements are parsed into clean document buffers entirely in your browser.

2

Document Chunking

The text is split using your chosen strategy. Section-based chunking identifies logical divisions (e.g. Experience, Skills) using rules. Sliders let you customize fixed character boundaries and overlap margins to control context retention.

3

Local Similarity Matching

A local TF-IDF (Term Frequency-Inverse Document Frequency) index is built from vocabulary. The engine computes Jaccard overlaps and Cosine Vector Similarity to match Resume chunks to JD chunks instantly as you edit.

4

AI Judge & Suggestion Engine

When you click evaluate, the matched chunk indexes are reviewed by the selected LLM provider. The AI judge acts as an expert recruiter to generate category scores, prioritized gaps, and rewrite suggestions.

Interactive Score Recalculation Note

If Local Cosine Similarity or Keyword Overlap (TF-IDF) is selected as the Similarity Method, all matches, scores, and alignment maps update **instantly** as you change strategies, drag sliders, or edit raw text in the Chunk Playground. If AI Judge is selected, a new evaluation must be run to update the LLM metrics after changes.
